第一週 Day 1

Java 語言基礎與 OOP 筆記

  • var 是 Java 10 引入的區域型別推斷，不能用在 method signature 上。
  • this 指向當前物件，super 呼叫父類別成員。

record class 範例：

public record Person(String name, int age) {}

sealed class 範例（Java 15+）：

public sealed class Animal permits Dog, Cat {}
final class Dog extends Animal {}
non-sealed class Cat extends Animal {}

模擬考題

問題 1

int a = 10;
int b = 20;
var result = (a > b) ? "a is greater" : 10;
System.out.println(result);

這段程式碼會？

答案：B

因為 var 需要型別一致，而 "a is greater" 是 String，10 是 int。

問題 2

sealed class Vehicle permits Car, Truck {}
final class Car extends Vehicle {}
non-sealed class Truck extends Vehicle {}
class Bike extends Vehicle {}

這段程式碼會怎樣？

答案：B

Bike 沒有在 permits 列表裡，無法繼承 Vehicle。

問題 3

哪個不是合法的 Java 語句？

答案：A

因為 var 無法從 null 推斷型別。

第一週 Day 2

例外處理（Exception Handling）

基礎語法：

try {
    // 可能拋出例外的程式碼
} catch (IOException e) {
    // 處理 IOException
} finally {
    // 一定會執行的區塊
}

關鍵概念：

  • checked exception：必須處理（如 IOException, SQLException）
  • unchecked exception：執行期例外（如 NullPointerException, ArrayIndexOutOfBoundsException）
  • 多個 catch 順序要由小到大（子類別在前）
  • 可使用 multi-catch（Java 7+）

Multi-catch 範例：

try {
    // ...
} catch (IOException | SQLException e) {
    // 共同處理邏輯
}

練習題（例外處理）

問題 1

哪一個可以合法編譯？

A. catch (Exception e | IOException ex) { }
B. catch (IOException | SQLException e) { }
C. catch (Throwable | Error e) { }

答案：B

(A) 中 e 和 ex 是不同變數名，不合法。

(C) 中 Throwable 和 Error 是繼承關係，不能一起使用。

第二週 Day 1

泛型基礎與類別設計

筆記重點：

  • 泛型類別與方法
  • 型別參數：<T>, <K,V> 等用法
  • Raw type 的風險

範例：

class Box<T> {
    private T value;
    public void set(T value) { this.value = value; }
    public T get() { return value; }
}

練習題

問題 1

Box<String> box = new Box<>();
box.set("Hello");
System.out.println(box.get()); // 輸出為？

答案：Hello

這是泛型類別的基本用法，Box<String> 表示這個 box 專門裝 String，呼叫 set("Hello") 設值後，get() 就能正確取出。

第二週 Day 2

泛型通配字元（wildcard）

筆記重點：

  • ?, ? extends T, ? super T 差異
  • PECS 原則（Producer Extends, Consumer Super）

範例：

void print(List<? extends Number> list) {
    list.forEach(System.out::println);
}

練習題

問題 1

List<? super Integer> list = new ArrayList<Number>();
list.add(10); // 合法嗎？

答案：合法

? super Integer 代表接受 Integer 或其父類別（例如 Number）。可以寫入 Integer 值，但取出來的型別只會是 Object。

第二週 Day 3

泛型方法與類型推斷

筆記重點：

  • 泛型方法宣告 <T> T methodName(T t)
  • 類型推斷與限制條件（bounded type）

範例：

public <T extends Number> void printValue(T t) {
    System.out.println(t);
}

練習題

問題 1

<T extends Comparable<T>> T max(T a, T b) {
    return a.compareTo(b) > 0 ? a : b;
}

答案：合法，可以比較大小的泛型方法

這是帶有上限限制的泛型方法，T extends Comparable<T> 確保 T 能呼叫 compareTo() 方法，用來找最大值，常用於排序與選擇最大元素。

範例用法：

System.out.println(max("apple", "banana")); // banana

第二週 Day 5

常用函式介面

筆記重點：

介面 功能
Predicate<T> 判斷（回傳布林值）
Function<T,R> 轉換（輸入 T 回傳 R）
Consumer<T> 接收資料但無回傳值
Supplier<T> 不接受參數，提供資料

範例：

Function<String, Integer> f = s -> s.length();
System.out.println(f.apply("Hello")); // 5

練習題

問題 1

Supplier<Double> random = () -> Math.random();
System.out.println(random.get());

答案：輸出為 0.0～1.0 間的隨機數字

Supplier 是無參數、有回傳的函式介面。這裡每次 get() 都會取得新的隨機數字。

Stream API 流程操作

筆記重點：

  • filter(), map(), sorted(), distinct()（中間操作）

範例：

List<String> names = List.of("Tom", "Jerry", "Spike");

練習題

問題 1

List<String> words = List.of("apple", "banana", "kiwi");

答案：6

第三週 Day 1

Java 模組系統基礎

筆記重點：

  • Java 9 引入 Module System，提供封裝與依賴控制
  • module-info.java 是模組描述檔案

模組主要結構：

module my.module.name {
    exports com.myapp.api;
    requires java.sql;
}

練習題

問題 1

module com.shop {
    requires java.base;
    exports com.shop.core;
}

此模組可以使用哪些 Java 模組？requires java.base; 有必要嗎？

答案：A

所有模組自動依賴 java.base，可省略 requires java.base;。可使用 java.base 提供的所有類別（例如 String、System、List）。

第三週 Day 2

module-info.java 與依賴關係

筆記重點：

  • exports：允許其他模組使用某 package
  • requires：宣告所依賴的模組
  • opens：允許反射存取 package（用於框架）
  • requires transitive：間接依賴

範例：

module A {
    exports a.util;
}

module B {
    requires A;
    requires transitive C;
}

練習題

問題 1

module service {
    requires com.fasterxml.jackson.databind;
    opens com.service.internal to jackson.databind;
}

上述設定有何用處？為何使用 opens 而非 exports？

答案：A

opens 是為了讓 jackson.databind 透過反射讀取 com.service.internal 的 class。exports 是給其他模組 compile-time 使用，opens 是給 runtime（如反射）使用。

第四週 Day 1

資料庫連線與 JDBC 基礎

筆記重點：

  • Java Database Connectivity (JDBC) 使 Java 程式能與資料庫互動
  • 主要步驟：
  • 下載並載入資料庫驅動
  • 建立連線：DriverManager.getConnection()
  • 執行 SQL 查詢：Statement 或 PreparedStatement
  • 處理結果：ResultSet
  • 關閉連線、語句與結果集

範例：

Connection conn = DriverManager.getConnection("jdbc:mysql://localhost:3306/test", "root", "password");
Statement stmt = conn.createStatement();
ResultSet rs = stmt.executeQuery("SELECT * FROM users");
while (rs.next()) {
    System.out.println(rs.getString("name"));
}

練習題

問題 1

Connection conn = DriverManager.getConnection("jdbc:sqlite:sample.db");
PreparedStatement pstmt = conn.prepareStatement("SELECT * FROM employees WHERE salary > ?");
pstmt.setInt(1, 50000);
ResultSet rs = pstmt.executeQuery();
while (rs.next()) {
    System.out.println(rs.getString("name"));
}

這段程式會查詢什麼資料？

答案：A

查詢所有薪水大於 50000 的員工資料，並列印出每位員工的 name。

第四週 Day 2

JDBC 進階操作與事務處理

筆記重點：

  • 事務管理：commit(), rollback(), setAutoCommit(false)
  • 自訂錯誤處理，確保資料庫一致性

範例：

conn.setAutoCommit(false);
try {
    Statement stmt = conn.createStatement();
    stmt.executeUpdate("UPDATE accounts SET balance = balance - 100 WHERE user_id = 1");
    stmt.executeUpdate("UPDATE accounts SET balance = balance + 100 WHERE user_id = 2");
    conn.commit();
} catch (SQLException e) {
    conn.rollback();
}

練習題

問題 1

conn.setAutoCommit(false);
Statement stmt = conn.createStatement();
stmt.executeUpdate("INSERT INTO products (name, price) VALUES ('Laptop', 1200)");
conn.commit();

setAutoCommit(false) 的作用是什麼？

答案：A

停用自動提交，允許在事務中執行多個操作，只有當 commit() 被呼叫時才將變更提交到資料庫。

第四週 Day 4

序列化與反序列化

筆記重點：

  • 實作 Serializable 介面，使物件可轉換為 byte 流
  • ObjectOutputStream 與 ObjectInputStream 用於序列化與反序列化

範例：

class Person implements Serializable {
    private String name;
    private int age;
    // Constructor, getters, and setters
}

ObjectOutputStream out = new ObjectOutputStream(new FileOutputStream("person.ser"));
out.writeObject(person);
out.close();

ObjectInputStream in = new ObjectInputStream(new FileInputStream("person.ser"));
Person person = (Person) in.readObject();
in.close();

練習題

問題 1

class Animal implements Serializable {
    private String type;
    private int age;
}

Animal animal = new Animal();
ObjectOutputStream out = new ObjectOutputStream(new FileOutputStream("animal.ser"));
out.writeObject(animal);
out.close();

此段程式會將物件序列化為哪種格式？

答案：A

將 Animal 物件序列化為二進位格式，並儲存為 animal.ser 檔案。

範例：

try {
    FileInputStream file = new FileInputStream("file.txt");
} catch (FileNotFoundException e) {
    System.out.println("File not found");
} finally {
    System.out.println("Always executed");
}

練習題

問題 1

try {
    String s = null;
    System.out.println(s.length());
} catch (NullPointerException e) {
    System.out.println("Caught NullPointerException");
}

程式會輸出什麼？

答案：A

Caught NullPointerException，因為對 null 物件呼叫 length() 方法會拋出 NullPointerException。

第四週 Day 6

異常類型與自定義異常

筆記重點：

  • 自定義異常類別繼承 Exception 或 RuntimeException
  • 檢查型異常（checked exceptions）需要處理，非檢查型異常（unchecked exceptions）則不需要

範例：

class InvalidAgeException extends Exception {
    public InvalidAgeException(String message) {
        super(message);
    }
}

練習題

問題 1

class AgeValidator {
    public void validate(int age) throws InvalidAgeException {
        if (age < 18) {
            throw new InvalidAgeException("Age is below 18");
        }
    }
}

當 age < 18 時，程式會拋出什麼異常？

答案：A

InvalidAgeException，這是自定義的檢查型異常。

第四週 Day 7

效能分析與最佳化

筆記重點：

  • 使用 java.lang.management 來檢查 JVM 效能
  • 使用 -Xms 與 -Xmx 設定記憶體大小
  • 記憶體分析工具：JVisualVM、JProfiler、YourKit

範例：

long start = System.nanoTime();
for (int i = 0; i < 1000; i++) {
    // some code
}
long end = System.nanoTime();
System.out.println("Elapsed time: " + (end - start));

練習題

問題 1

long start = System.currentTimeMillis();
Thread.sleep(2000);
long end = System.currentTimeMillis();
System.out.println("Elapsed time: " + (end - start) + " milliseconds");

此程式計算的是什麼時間？

答案：A

計算程式執行 2 秒鐘（2000 毫秒）的經過時間。
