第一週 Day 1

Java 語言基礎與 OOP 筆記

  • var 是 Java 10 引入的區域型別推斷，不能用在 method signature 上。
  • this 指向當前物件，super 呼叫父類別成員。

record class 範例：

public record Person(String name, int age) {}

sealed class 範例（Java 15+）：

public sealed class Animal permits Dog, Cat {}
final class Dog extends Animal {}
non-sealed class Cat extends Animal {}

模擬考題

問題 1

int a = 10;
int b = 20;
var result = (a > b) ? "a is greater" : 10;
System.out.println(result);

這段程式碼會？

答案：B

因為 var 需要型別一致，而 "a is greater" 是 String，10 是 int。

問題 2

sealed class Vehicle permits Car, Truck {}
final class Car extends Vehicle {}
non-sealed class Truck extends Vehicle {}
class Bike extends Vehicle {}

這段程式碼會怎樣？

答案：B

Bike 沒有在 permits 列表裡，無法繼承 Vehicle。

問題 3

哪個不是合法的 Java 語句？

答案：A

因為 var 無法從 null 推斷型別。

第一週 Day 2

例外處理（Exception Handling）

基礎語法：

try {
    // 可能拋出例外的程式碼
} catch (IOException e) {
    // 處理 IOException
} finally {
    // 一定會執行的區塊
}

關鍵概念：

  • checked exception：必須處理（如 IOException, SQLException）
  • unchecked exception：執行期例外（如 NullPointerException, ArrayIndexOutOfBoundsException）
  • 多個 catch 順序要由小到大（子類別在前）
  • 可使用 multi-catch（Java 7+）

Multi-catch 範例：

try {
    // ...
} catch (IOException | SQLException e) {
    // 共同處理邏輯
}

練習題（例外處理）

問題 1

哪一個可以合法編譯？

A. catch (Exception e | IOException ex) { }
B. catch (IOException | SQLException e) { }
C. catch (Throwable | Error e) { }

答案：B

(A) 中 e 和 ex 是不同變數名，不合法。

(C) 中 Throwable 和 Error 是繼承關係，不能一起使用。

class Box<T> {

Box<String> box = new Box<>();

答案：Hello

第二週 Day 2

泛型通配字元（wildcard）

筆記重點：

  • ?, ? extends T, ? super T 差異
  • PECS 原則（Producer Extends, Consumer Super）

範例：

void print(List<? extends Number> list) {
    list.forEach(System.out::println);
}

練習題

問題 1

List<? super Integer> list = new ArrayList<Number>();
list.add(10); // 合法嗎？

答案：合法

? super Integer 代表接受 Integer 或其父類別（例如 Number）。可以寫入 Integer 值，但取出來的型別只會是 Object。

第二週 Day 3

泛型方法與類型推斷

筆記重點：

  • 泛型方法宣告 <T> T methodName(T t)
  • 類型推斷與限制條件（bounded type）

範例：

public <T extends Number> void printValue(T t) {
    System.out.println(t);
}

練習題

問題 1

<T extends Comparable<T>> T max(T a, T b) {
    return a.compareTo(b) > 0 ? a : b;
}

答案：合法，可以比較大小的泛型方法

這是帶有上限限制的泛型方法，T extends Comparable<T> 確保 T 能呼叫 compareTo() 方法，用來找最大值，常用於排序與選擇最大元素。

範例用法：

System.out.println(max("apple", "banana")); // banana

第二週 Day 5

常用函式介面

筆記重點：

介面 功能
Predicate<T> 判斷（回傳布林值）
Function<T,R> 轉換（輸入 T 回傳 R）
Consumer<T> 接收資料但無回傳值
Supplier<T> 不接受參數，提供資料

範例：

Function<String, Integer> f = s -> s.length();
System.out.println(f.apply("Hello")); // 5

練習題

問題 1

Supplier<Double> random = () -> Math.random();
System.out.println(random.get());

答案：輸出為 0.0～1.0 間的隨機數字

Supplier 是無參數、有回傳的函式介面。這裡每次 get() 都會取得新的隨機數字。

第三週 Day 2

module-info.java 與依賴關係

筆記重點：

  • exports：允許其他模組使用某 package
  • requires：宣告所依賴的模組
  • opens：允許反射存取 package（用於框架）
  • requires transitive：間接依賴

範例：

module A {
    exports a.util;
}

module B {
    requires A;
    requires transitive C;
}

練習題

問題 1

module service {
    requires com.fasterxml.jackson.databind;
    opens com.service.internal to jackson.databind;
}

上述設定有何用處？為何使用 opens 而非 exports？

答案：A

opens 是為了讓 jackson.databind 透過反射讀取 com.service.internal 的 class。exports 是給其他模組 compile-time 使用，opens 是給 runtime（如反射）使用。

練習題

問題 1：JDBC 批次處理

以下關於 JDBC 批次處理的敘述，何者正確？

批次處理可以減少資料庫往返次數，提高效能。它可以用於任何 SQL 操作，不一定要在交易中執行，且可以返回每個操作的影響行數。

JDBC 進階操作

交易管理

  • 交易隔離等級
  • 交易提交與回滾
  • Savepoint 的使用

交易管理範例

conn.setAutoCommit(false);
try {
    // 執行多個 SQL 操作
    stmt1.executeUpdate("UPDATE accounts SET balance = balance - 100 WHERE id = 1");
    stmt2.executeUpdate("UPDATE accounts SET balance = balance + 100 WHERE id = 2");
    conn.commit();
} catch (SQLException e) {
    conn.rollback();
    throw e;
}

ResultSet 進階操作

  • 可滾動的 ResultSet
  • 可更新的 ResultSet
  • ResultSet 元數據

可滾動 ResultSet 範例

Statement stmt = conn.createStatement(
    ResultSet.TYPE_SCROLL_INSENSITIVE,
    ResultSet.CONCUR_READ_ONLY
);
ResultSet rs = stmt.executeQuery("SELECT * FROM users");
rs.last();  // 移動到最後一行
rs.previous();  // 移動到前一行
rs.absolute(5);  // 移動到第 5 行
